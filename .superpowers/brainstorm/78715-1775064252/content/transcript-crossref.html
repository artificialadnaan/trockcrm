Gaps Found — Need to Add to Spec

1. Project Type / Business Line Categories Not Defined
Adam explicitly listed the four buckets: Restoration, Multifamily, Commercial (new con/land dev), Service. And sub-types within Multifamily: Student Housing, Senior Living, Traditional Multifamily. Spec has project_type VARCHAR(100) but doesn't define the allowed values or the parent→child relationship for reporting ("show me the pie chart of those different buckets").
Fix: Add public.project_type_config table with parent/child hierarchy. Locked dropdown values. Reports group by parent business line.
2. Region Tracking Not Defined
Adam mentioned "Texas region or East Coast region" as dropdown options. The spec has property_state but no region concept. Regions group states for reporting and filtering.
Fix: Add public.region_config table and region_id FK on deals. Predefined regions with state mappings.
3. Ticketing System / Feature Request Portal Not Mentioned
Adnaan mentioned creating a ticketing system for issues and a feature request portal. These are external tools (not CRM features), but the spec should note their existence as part of the support infrastructure.
Fix: Add note in Section 22 (Documentation) referencing the support ticketing system and feature request portal for post-launch.
4. Call Logging UI Not Explicitly Detailed
Meeting emphasized logging calls with notes, duration, outcome. Spec has activities table with outcome field (added in cross-ref fix), but no explicit mention of a call logging form UI as a key component in Section 11. Build tracker has it as s4-9.
Fix: Add call/note/meeting logging forms to Key Components in Section 11.
5. Default Date Filter Behavior
Multiple people complained: "every time I pop open, it literally shows me numbers since the day we started HubSpot... not from this year". The spec doesn't specify default date range behavior for dashboards and reports.
Fix: Spec should state: all dashboards and reports default to current calendar year. Pipeline views default to active deals only. No "since inception" defaults.
