Database Schema — Per-Office Tables

These tables exist in each office_{slug} schema (e.g., office_dallas.deals, office_houston.deals)

Deals & Pipeline

office_{slug}.deals
id UUID PK DEFAULT gen_random_uuid()
deal_number VARCHAR(50) UNIQUE NOT NULL -- auto-generated or manual
name VARCHAR(500) NOT NULL
stage_id UUID FK → public.pipeline_stage_config.id NOT NULL
assigned_rep_id UUID FK → public.users.id NOT NULL
primary_contact_id UUID FK → contacts.id
-- Estimate Values (flat columns)
dd_estimate NUMERIC(14,2) -- Due Diligence estimate
bid_estimate NUMERIC(14,2) -- bid/proposal amount
awarded_amount NUMERIC(14,2) -- contract awarded value
current_contract_value NUMERIC(14,2) GENERATED ALWAYS AS (COALESCE(awarded_amount, 0) + change_order_total) STORED
change_order_total NUMERIC(14,2) DEFAULT 0 -- denormalized SUM from change_orders table, updated by trigger
-- Metadata
description TEXT
property_address TEXT
property_city VARCHAR(255)
property_state VARCHAR(2)
property_zip VARCHAR(10)
project_type VARCHAR(100) -- "Commercial", "Industrial", "Healthcare", etc.
source VARCHAR(100) -- "Bid Board", "Referral", "Cold Call", etc.
-- Procore Integration
procore_project_id BIGINT -- linked Procore project (NULL until won)
procore_bid_id BIGINT -- linked Procore Bid Board item
procore_last_synced_at TIMESTAMPTZ
-- Lost Deal
lost_reason_id UUID FK → public.lost_deal_reasons.id -- required when stage = Closed Lost
lost_notes TEXT -- required free-text when stage = Closed Lost
lost_at TIMESTAMPTZ
-- Dates & Lifecycle
expected_close_date DATE
actual_close_date DATE
last_activity_at TIMESTAMPTZ -- updated on any deal activity (email, note, call)
stage_entered_at TIMESTAMPTZ NOT NULL DEFAULT NOW() -- reset on stage change, used for stale alerts
is_active BOOLEAN DEFAULT true
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
-- HubSpot Migration
hubspot_deal_id VARCHAR(50) -- preserved for traceability, NULL for new deals
office_{slug}.deal_stage_history
-- Written by PG trigger on deals.stage_id change
id UUID PK DEFAULT gen_random_uuid()
deal_id UUID FK → deals.id NOT NULL
from_stage_id UUID FK → public.pipeline_stage_config.id
to_stage_id UUID FK → public.pipeline_stage_config.id NOT NULL
changed_by UUID FK → public.users.id NOT NULL
is_backward_move BOOLEAN DEFAULT false
is_director_override BOOLEAN DEFAULT false
override_reason TEXT -- required when is_director_override = true
duration_in_previous_stage INTERVAL -- computed from deal.stage_entered_at
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
office_{slug}.change_orders
id UUID PK DEFAULT gen_random_uuid()
deal_id UUID FK → deals.id NOT NULL
co_number INTEGER NOT NULL -- sequential per deal
title VARCHAR(500) NOT NULL
amount NUMERIC(14,2) NOT NULL -- positive = addition, negative = deduction
status ENUM('pending','approved','rejected') DEFAULT 'pending'
procore_co_id BIGINT -- linked Procore change order
approved_at TIMESTAMPTZ
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
UNIQUE(deal_id, co_number)
office_{slug}.deal_approvals
-- Tracks required approvals for stage gates
id UUID PK DEFAULT gen_random_uuid()
deal_id UUID FK → deals.id NOT NULL
target_stage_id UUID FK → public.pipeline_stage_config.id NOT NULL
requested_by UUID FK → public.users.id NOT NULL
approved_by UUID FK → public.users.id
status ENUM('pending','approved','rejected') DEFAULT 'pending'
notes TEXT
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
resolved_at TIMESTAMPTZ

Edge Cases Handled

  • DD separation: pipeline_stage_config.is_active_pipeline = false for DD stage — reporting queries filter on this flag to exclude DD from active pipeline totals
  • Backward move prevention: API compares display_order of current vs target stage. If target < current AND user role != director → reject. Logged in deal_stage_history
  • Closed Lost enforcement: API requires lost_reason_id + lost_notes when target stage = Closed Lost. PG CHECK constraint as safety net
  • Change order denormalization: PG trigger on change_orders INSERT/UPDATE/DELETE recalculates deals.change_order_total = SUM of approved COs
  • Generated column: current_contract_value is always awarded_amount + change_order_total — can never be stale
  • Stale deal detection: Worker compares NOW() - stage_entered_at against pipeline_stage_config.stale_threshold_days
  • HubSpot traceability: hubspot_deal_id preserved on migrated deals so team can cross-reference during validation
