Enhanced File Organization & Searchability

Updated files table with tagging, folder structure, full-text search, and naming conventions

office_{slug}.files (enhanced)
id UUID PK DEFAULT gen_random_uuid()
-- Classification & Organization
category ENUM('photo','contract','rfp','estimate','change_order','proposal','permit','inspection','correspondence','insurance','warranty','closeout','other') NOT NULL
subcategory VARCHAR(100) -- "Site Visit", "Progress", "Final Walkthrough", "Damage", etc.
folder_path VARCHAR(1000) -- virtual folder: "Estimates/Revisions" or "Photos/Site Visits/2026-04"
tags TEXT[] DEFAULT '{}' -- ["exterior", "roof", "before-photo", "client-approved"]
display_name VARCHAR(500) NOT NULL -- human-readable name, auto-generated or user-set
-- Naming Convention (auto-generated)
system_filename VARCHAR(500) NOT NULL -- "{DealNumber}_{Category}_{Date}_{Seq}.{ext}"
original_filename VARCHAR(500) NOT NULL -- what the user uploaded
-- File Details
mime_type VARCHAR(100) NOT NULL
file_size_bytes BIGINT NOT NULL
file_extension VARCHAR(20) NOT NULL -- ".pdf", ".jpg", ".xlsx"
r2_key VARCHAR(1000) UNIQUE NOT NULL
r2_bucket VARCHAR(100) NOT NULL
-- Associations
deal_id UUID FK → deals.id
contact_id UUID FK → contacts.id
procore_project_id BIGINT
change_order_id UUID FK → change_orders.id -- for CO-specific docs
-- Metadata & Search
description TEXT
notes TEXT -- internal notes about this file
version INTEGER DEFAULT 1 -- for revised documents (Estimate v1, v2, v3)
parent_file_id UUID FK → files.id -- links revisions to original
taken_at TIMESTAMPTZ -- photo EXIF timestamp
geo_lat NUMERIC(10,7) -- photo GPS from EXIF
geo_lng NUMERIC(10,7)
-- Full-Text Search
search_vector TSVECTOR GENERATED ALWAYS AS (
setweight(to_tsvector('english', COALESCE(display_name, '')), 'A') ||
setweight(to_tsvector('english', COALESCE(description, '')), 'B') ||
setweight(to_tsvector('english', COALESCE(notes, '')), 'C') ||
setweight(to_tsvector('english', COALESCE(array_to_string(tags, ' '), '')), 'B')
) STORED
-- Standard
uploaded_by UUID FK → public.users.id NOT NULL
is_active BOOLEAN DEFAULT true
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
INDEX gin ON (search_vector) -- full-text search
INDEX gin ON (tags) -- tag filtering
INDEX ON (deal_id, category, created_at DESC) -- deal file browser
INDEX ON (folder_path, display_name) -- folder navigation
INDEX ON (category, file_extension) -- filter by type
CHECK (deal_id IS NOT NULL OR contact_id IS NOT NULL OR procore_project_id IS NOT NULL)

Pattern: {DealNumber}_{Category}_{YYYY-MM-DD}_{Seq}.{ext}

Search & Organization Features

  • Full-text search: PG tsvector with weighted fields — display name (A), description + tags (B), notes (C). Search "roof damage exterior" finds relevant photos instantly.
  • Tag system: Freeform tags with GIN index. Filter by tags across deals. Suggest existing tags on upload for consistency.
  • Virtual folders: folder_path creates a navigable hierarchy without physical directories. Default folders auto-created per deal, custom folders allowed.
  • Version tracking: parent_file_id + version chains document revisions. "Estimate v3" links back to v1 and v2. UI shows latest by default with version history expandable.
  • Auto-naming: Consistent {DealNumber}_{Category}_{Date}_{Seq} pattern. Original filename preserved separately. Downloadable files use system name for consistency across the team.
  • R2 key hierarchy: Mirrors the virtual folder structure. Browsable in Cloudflare R2 console if needed. Office-scoped at the top level.
  • EXIF extraction: GPS coordinates and timestamps pulled from photo metadata on upload. Powers the photo timeline and potential future map views.
  • Category-specific filters: "Show me all contracts across all deals" or "All site visit photos from March 2026" — indexed for fast queries.
