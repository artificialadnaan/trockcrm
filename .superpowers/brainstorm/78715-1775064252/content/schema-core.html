Database Schema — Core Tables

Schema-per-office architecture: public schema for global data, office_{slug} schemas for tenant data. PG triggers on all audited tables.

Global Schema (public)

Shared across all offices — auth, config, office registry

public.offices
id UUID PK DEFAULT gen_random_uuid()
name VARCHAR(255) NOT NULL -- "Dallas", "Houston"
slug VARCHAR(100) UNIQUE NOT NULL -- "dallas", "houston" (used for schema name)
address TEXT
phone VARCHAR(20)
is_active BOOLEAN DEFAULT true
settings JSONB DEFAULT '{}' -- stale deal thresholds, notification prefs
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
public.users
id UUID PK DEFAULT gen_random_uuid()
email VARCHAR(255) UNIQUE NOT NULL -- MS Entra email
display_name VARCHAR(255) NOT NULL
azure_ad_id VARCHAR(255) UNIQUE -- from MS Entra token
avatar_url TEXT
role ENUM('admin','director','rep') NOT NULL
office_id UUID FK → offices.id NOT NULL
is_active BOOLEAN DEFAULT true
notification_prefs JSONB DEFAULT '{}'
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
public.user_office_access
-- Directors/admins who can access multiple offices
id UUID PK DEFAULT gen_random_uuid()
user_id UUID FK → users.id NOT NULL
office_id UUID FK → offices.id NOT NULL
role_override ENUM('admin','director','rep') -- NULL = use user's default role
UNIQUE(user_id, office_id)
public.pipeline_stage_config
-- Defines allowed stages and their order, requirements, and stale thresholds
id UUID PK DEFAULT gen_random_uuid()
name VARCHAR(100) NOT NULL -- "DD", "Estimating", "Bid Sent", etc.
slug VARCHAR(100) UNIQUE NOT NULL
display_order INTEGER NOT NULL
is_active_pipeline BOOLEAN DEFAULT true -- false for DD (excluded from pipeline totals)
is_terminal BOOLEAN DEFAULT false -- true for Closed Won, Closed Lost
required_fields JSONB DEFAULT '[]' -- ["dd_estimate", "site_photos"] fields required to enter this stage
required_documents JSONB DEFAULT '[]' -- ["rfp", "estimate"] document types required
required_approvals JSONB DEFAULT '[]' -- ["director"] roles that must approve
stale_threshold_days INTEGER -- NULL = no stale alert for this stage
procore_stage_mapping VARCHAR(100) -- maps to Procore project status
color VARCHAR(7) -- hex color for UI badges
public.lost_deal_reasons
id UUID PK DEFAULT gen_random_uuid()
label VARCHAR(255) NOT NULL -- "Price", "Timing", "Went with competitor", etc.
is_active BOOLEAN DEFAULT true
display_order INTEGER NOT NULL

Design Notes

  • Users are global — a user belongs to a primary office but can access others via user_office_access. Admins see all offices.
  • Pipeline config is global — all offices share the same stage definitions. Per-office stage customization is out of scope (SOW says "T Rock-specific stages").
  • Settings JSONB — stale deal thresholds can be per-office (in offices.settings) or global (in pipeline_stage_config.stale_threshold_days). Per-office overrides win.
  • Soft deletes — is_active flags instead of hard deletes. Audit trail requires we never lose records.
