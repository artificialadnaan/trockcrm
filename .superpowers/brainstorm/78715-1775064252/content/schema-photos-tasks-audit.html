Database Schema — Photos, Tasks, Notifications & Audit

Per-office schema: file management, automated task system, real-time alerts, and field-level audit trail

Photos & Documents

office_{slug}.files
-- Unified table for photos, documents, contracts, RFPs, estimates
id UUID PK DEFAULT gen_random_uuid()
category ENUM('photo','document','contract','rfp','estimate','change_order','other') NOT NULL
filename VARCHAR(500) NOT NULL
original_filename VARCHAR(500) NOT NULL -- user's original filename
mime_type VARCHAR(100) NOT NULL
file_size_bytes BIGINT NOT NULL
r2_key VARCHAR(1000) UNIQUE NOT NULL -- Cloudflare R2 object key
r2_bucket VARCHAR(100) NOT NULL
-- Associations (polymorphic — at least one required)
deal_id UUID FK → deals.id
contact_id UUID FK → contacts.id
procore_project_id BIGINT -- for project-level photos not tied to a deal
-- Metadata
description TEXT
taken_at TIMESTAMPTZ -- photo timestamp (from EXIF or manual)
uploaded_by UUID FK → public.users.id NOT NULL
is_active BOOLEAN DEFAULT true
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
INDEX ON (deal_id, category, created_at DESC) -- deal photo timeline
CHECK (deal_id IS NOT NULL OR contact_id IS NOT NULL OR procore_project_id IS NOT NULL)

Tasks & Alerts

office_{slug}.tasks
id UUID PK DEFAULT gen_random_uuid()
title VARCHAR(500) NOT NULL
description TEXT
type ENUM('follow_up','stale_deal','inbound_email','approval_request','touchpoint','manual','system') NOT NULL
priority ENUM('urgent','high','normal','low') DEFAULT 'normal'
status ENUM('pending','in_progress','completed','dismissed') DEFAULT 'pending'
-- Assignment
assigned_to UUID FK → public.users.id NOT NULL
created_by UUID FK → public.users.id -- NULL for system-generated tasks
-- Associations
deal_id UUID FK → deals.id
contact_id UUID FK → contacts.id
email_id UUID FK → emails.id -- for inbound_email type
-- Scheduling
due_date DATE
due_time TIME
remind_at TIMESTAMPTZ
completed_at TIMESTAMPTZ
is_overdue BOOLEAN GENERATED ALWAYS AS (due_date < CURRENT_DATE AND status IN ('pending','in_progress')) STORED
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
INDEX ON (assigned_to, status, due_date) -- daily task list query
INDEX ON (assigned_to, status, priority) -- prioritized view
office_{slug}.notifications
-- SSE-delivered notifications, persisted for notification center
id UUID PK DEFAULT gen_random_uuid()
user_id UUID FK → public.users.id NOT NULL
type ENUM('stale_deal','inbound_email','task_assigned','approval_needed','activity_drop','deal_won','deal_lost','stage_change','system') NOT NULL
title VARCHAR(500) NOT NULL
body TEXT
link VARCHAR(1000) -- deep link to relevant CRM page
is_read BOOLEAN DEFAULT false
read_at TIMESTAMPTZ
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
INDEX ON (user_id, is_read, created_at DESC) -- unread notifications query

Audit Trail

office_{slug}.audit_log
-- Written by PG triggers on all audited tables. APPEND-ONLY — no updates or deletes.
id BIGSERIAL PK -- bigint for high volume
table_name VARCHAR(100) NOT NULL
record_id UUID NOT NULL
action ENUM('insert','update','delete') NOT NULL
changed_by UUID -- set via SET app.current_user_id before queries
changes JSONB NOT NULL -- { "field": { "old": "X", "new": "Y" } } for updates
full_row JSONB -- full row snapshot on insert/delete
ip_address INET
user_agent VARCHAR(500)
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
INDEX ON (table_name, record_id, created_at DESC) -- record history
INDEX ON (changed_by, created_at DESC) -- user activity audit
INDEX ON (created_at) -- time-based queries, partition candidate

Reporting & Saved Filters

public.saved_reports
-- Global table: locked company reports + user custom reports
id UUID PK DEFAULT gen_random_uuid()
name VARCHAR(255) NOT NULL
entity ENUM('deals','contacts','activities','tasks') NOT NULL
config JSONB NOT NULL -- { filters: [], columns: [], sort: {}, chart_type: "" }
is_locked BOOLEAN DEFAULT false -- true = company-wide preset, users can't edit
is_default BOOLEAN DEFAULT false -- shows on dashboard by default
created_by UUID FK → public.users.id -- NULL for system presets
office_id UUID FK → public.offices.id -- NULL = available to all offices
visibility ENUM('private','office','company') DEFAULT 'private'
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

Edge Cases Handled

  • Photo timeline ordering: taken_at (EXIF timestamp) used for chronological display, falls back to created_at if no EXIF data
  • File orphan prevention: CHECK constraint requires at least one association (deal, contact, or procore project). No floating files.
  • Cross-team photo sharing: File is uploaded once, associated to a deal. Sales, estimating, and PM all see it via the deal record — SOW requirement met without duplication.
  • R2 key uniqueness: Keys use {office_slug}/{entity}/{id}/{uuid}-{original_filename} pattern — globally unique, human-readable in R2 console.
  • Task auto-generation: Worker's daily cron generates tasks by scanning: overdue follow-ups (deals with last_activity_at > threshold), inbound emails (unprocessed), stale deals (stage_entered_at > threshold). Each task type has its own priority weighting.
  • is_overdue generated column: Postgres computes this — no application logic needed. Task list queries filter on it directly.
  • Audit append-only: No UPDATE or DELETE grants on audit_log. PG trigger writes only. Partitionable by created_at for long-term storage.
  • Audit user tracking: SET LOCAL app.current_user_id = '...' in every transaction. PG trigger reads current_setting('app.current_user_id'). Middleware sets this automatically.
  • Saved reports isolation: Locked reports (is_locked=true) can't be modified by users. Custom reports are private by default, can be shared to office or company level.
  • Report config JSON: Structured as { filters, columns, sort, chart_type } — same shape scales to a visual query builder in the future.
